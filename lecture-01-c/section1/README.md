# 🎓 Section 1: Foundations of C & Developer Tooling

This directory contains my progress and solutions from **Section 1** of Harvard's CS50x. This section bridged the gap between visual logic (Scratch) and low-level programming (C), focusing on syntax, manual memory management, and professional development workflows.

## 🚀 Projects

### 📇 Contacts
A command-line application designed to practice multi-type data handling and formatted output.
* **Key Learning:** Using `get_string` and `get_int` from the CS50 library.
* **Concepts:** Variable declaration, static typing, and `printf` format specifiers (`%s`, `%i`).
* **Usage:**
  ```bash
  make contacts
  ./contacts
  ```

## 🧠 Key Technical Concepts

During this section, I mastered the following core programming constructs:

### 🛠️ The CS50 Toolchain
I integrated professional debugging and styling tools into my workflow:
* **`debug50`**: Using breakpoints and step-by-step execution to trace logical errors.
* **`style50`**: Enforcing clean code standards (indentation, naming conventions).
* **`check50`**: Automated unit testing for edge-case validation.

### 🚦 Control Flow & Logic
* **Conditionals**: Implementing `if/else` blocks for decision-making.
* **Loops**: 
    * `for`: For precise, counted iterations.
    * `while`: For condition-based repetition.
    * `do-while`: For guaranteed single execution (essential for **Input Validation**).

### 🏗️ Functional Abstraction
Moving beyond the `main` function, I practiced breaking down complex problems into modular, reusable custom functions.
* **Example:** Creating a `print_row(int n)` function to simplify the Mario pyramid logic.
